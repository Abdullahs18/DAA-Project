Algorithm Reference

Closest Pair of Points

Complexity: O(n log n)
Strategy: Divide & Conquer

1. Sort points by x-coordinate
2. Recursively find closest pair in left and right halves
3. Let δ = min(left_dist, right_dist)
4. Check strip of width 2δ around dividing line
5. Return overall minimum distance

Key insight: At most 6 points in any δ×2δ strip rectangle, so strip check is O(n).

Karatsuba Multiplication

Complexity: O(n^1.585)
Strategy: Divide & Conquer

Given x, y with n digits, split at midpoint m:
x = a·10^m + b, y = c·10^m + d

Compute only 3 multiplications:
ac = karatsuba(a, c)
bd = karatsuba(b, d)
ad+bc = karatsuba(a+b, c+d) − ac − bd

Result: ac·10^2m + (ad+bc)·10^m + bd

Brute force requires 4 multiplications vs 3 → T(n) = 3T(n/2) → O(n^log₂3)
